Design system mobile

MOS! premium dashboard para rotina alimentar.

Um sistema visual para comida, água, treino, plano, suplementação e medidas. Simples, forte, escaneável e sem cara de app fitness genérico.

Tokens de cor

Paleta curta, uso rigoroso.

A base é neutra. O verde é ação/assinatura. O roxo é auxiliar raro. Nenhuma outra cor entra no sistema.

Cor / Fundo / Sage #E3E5D6

Ambiente, superfície base, respiro e telas de baixa tensão.

Cor / Primária / Preto #000000

Tipografia, bordas, ações principais e painéis premium.

Cor / Acento / Lima #DAFF00

Uma vez por tela: progresso, conquista, CTA ou foco do momento.

Cor / Acento / Íris #766CFF

Secundário e raro: insight, comparação ou detalhe de apoio.

Implementação recomendada

HTML + SVG para gráficos dinâmicos

Use SVG para linhas, barras, áreas e sparklines. Os pontos, paths e larguras podem ser calculados com dados reais do app. Use CSS gradients para fundos e cards especiais. Evite imagens estáticas para gráficos.

Padrões do app

O que cada área precisa.

Cada fluxo deve ter uma composição clara: cabeçalho, dado principal, lista ou formulário, CTA principal e feedback.

Comida

Lista + detalhe + cérebro MOS

Cadastro com alimento e quantidade. Macros aparecem depois do cálculo.

Água

Meta + atalhos + histórico

Ação rápida, progresso evidente e remoção simples.

Treino

Lista + edição + execução

Estado atual, próximo exercício, conclusão e pausa sem confusão.

Plano

Abas fixas

Plano alimentar em lista; suplementação com dose e progresso.

Medidas

Cards empilhados

Evitar colunas apertadas. Gráfico sempre full-width no mobile.

Notificações

Toast + central

Mensagem curta, ação clara, sem bloquear o fluxo sem necessidade.

Regras finais

Checklist antes de implementar.

Use isso como revisão antes de mandar qualquer frame para implementação.

Fazer

  • Usar verde uma vez por tela como momento de destaque.
  • Manter botões e cards com função clara.
  • Preferir ícones para editar/apagar.
  • Usar H1 forte e body curto.
  • Empilhar gráficos no mobile.
  • Nomear componentes como Button / Primary.

Evitar

  • Vários cards competindo visualmente.
  • Texto longo dentro de cards compactos.
  • Roxo como cor dominante.
  • Sombras exageradas.
  • Botões sem feedback visual.
  • Layout com cara de academia genérica.
